### Intuition

---

- `n`보다 큰 자연수 중에서 이진수로 변환했을 때 `1`의 개수가 같은 가장 작은 수를 찾아야 한다.
- 숫자가 커질수록 답 후보가 되므로, `n + 1`부터 차례대로 확인하면 가장 먼저 조건을 만족하는 수가 정답이다.
- 이진수의 `1` 개수는 직접 문자열을 만들 수도 있지만, Java에서는 `Integer.bitCount()`로 더 간단하게 구할 수 있다.
- 문제의 핵심 조건은 숫자의 크기 자체가 아니라 이진수에서 켜져 있는 비트의 개수이다.

<br>

### Approach

---

- 먼저 `Integer.bitCount(n)`으로 원래 숫자 `n`의 `1` 개수를 `target`에 저장한다.
- `n + 1`, `n + 2`, ... 순서로 다음 숫자들을 확인한다.
- 각 후보 숫자 `next`에 대해 `Integer.bitCount(next)`를 구한다.
- 후보 숫자의 `1` 개수가 `target`과 같으면 조건을 만족하는 가장 작은 수이므로 `answer`에 저장하고 반복을 종료한다.
- 문자열 변환 방식도 가능하지만, `Integer.bitCount()`를 사용하면 이진 문자열 생성과 문자열 치환을 하지 않아도 된다.

<br>

### Complexity

---

- Time complexity: `O(k)`
  - `k`는 `n + 1`부터 정답을 찾을 때까지 확인한 후보 숫자의 개수이다.
  - 각 후보마다 `Integer.bitCount()`를 호출한다.
  - Java의 `int`는 32비트로 고정되어 있으므로 `Integer.bitCount()`는 상수 시간으로 볼 수 있다.

    (알고리즘 시간복잡도에서는 입력 크기와 무관하게 고정된 값이면 보통 상수 시간 O(1)으로 본다. 그 고정값이 32든, 64든, 1,000이든 입력이 커져도 변하지 않으면 점근적으로는 상수이다.)

- Space complexity: `O(1)`
  - 후보 숫자와 `1` 개수만 저장하므로 추가 공간은 상수이다.

<br>

### Code 1 (Java)

---

```java
class Solution {
    public int solution(int n) {
        int answer = 0;
        int MAX_N = 1000000;
        int target = Integer.bitCount(n);

        for (int i = 1; i <= MAX_N; i++) {
            int next = n + i;

            if (Integer.bitCount(next) == target) {
                answer = next;
                break;
            }
        }

        return answer;
    }
}
```

### Code 2 (Java)

```java
class Solution {
    public int solution(int n) {
        int answer = 0;
        int MAX_N = 1000000;
        int target = Integer.bitCount(n);

        String nBin = Integer.toBinaryString(n);
        int nCount = nBin.length() - nBin.replace("1", "").length();

        while (n <= MAX_N) {
            int next = n + 1;

            String nextBin = Integer.toBinaryString(next);
            int nextCount = nextBin.length() - nextBin.replace("1", "").length();

            if (nCount == nextCount) {
                answer = next;
                break;
            } else {
                n += 1;
            }
        }

        return answer;
    }
}
```

- 이진 문자열 생성과 문자열 치환한 버전이다.

### Better (Java)

```java
class Solution {
    public int solution(int n) {
        int smallest = n & -n;
        int ripple = n + smallest;
        int ones = n ^ ripple;

        ones = (ones >> 2) / smallest;

        return ripple | ones;
    }
}
```

- `smallest`는 `n`에서 가장 오른쪽에 있는 `1` 비트만 남긴 값이다.
- `ripple`은 가장 오른쪽의 연속된 `1` 묶음을 다음 자리로 올린 값이다.
- `ones`는 이동 과정에서 사라진 `1`들을 가능한 가장 오른쪽에 다시 배치하기 위한 값이다.
- 후보를 하나씩 증가시키지 않고 다음 수를 비트 연산으로 바로 만들 수 있다.
- 시간 복잡도는 고정된 `int` 범위에서는 `O(1)`, 추가 공간 복잡도도 `O(1)`이다.
