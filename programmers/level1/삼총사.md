<br>

### Intuition

---

- 학생 번호 배열에서 서로 다른 3명을 골랐을 때 합이 `0`이 되는 경우의 수를 구해야 한다.
- 배열의 길이가 최대 `13`이므로 가능한 3명 조합을 모두 확인해도 경우의 수가 많지 않다.
- 순서가 다른 같은 조합을 중복해서 세면 안 되므로, 인덱스를 `i < j < k` 형태로 고정해서 탐색한다.
- 별도의 정렬이나 자료구조 없이 3중 반복문으로 모든 조합을 확인하는 완전탐색이 적합하다.

<br>

### Approach

---

- 배열의 길이를 `n`에 저장한다.
- 첫 번째 학생 인덱스 `i`를 `0`부터 `n - 1`까지 순회한다.
- 두 번째 학생 인덱스 `j`는 `i + 1`부터 시작해, 이미 선택한 학생과 중복되지 않게 한다.
- 세 번째 학생 인덱스 `k`는 `j + 1`부터 시작해 항상 `i < j < k`가 되도록 한다.
- 세 학생의 번호 합이 `0`이면 삼총사를 만들 수 있으므로 `answer`를 `1` 증가시킨다.
- 모든 조합을 확인한 뒤 `answer`를 반환한다.

<br>

### Complexity

---

- Time complexity: `O(n³)`
  - 서로 다른 3명을 고르는 모든 조합을 3중 반복문으로 확인한다.

- Space complexity: `O(1)`
  - 입력 배열 외에 사용하는 추가 공간은 `answer`, `n`, `sum` 정도의 상수 공간이다.

<br>

### Code (Java)

---

```java
class Solution {
    public int solution(int[] number) {
        int answer = 0;
        int n = number.length;

        for (int i = 0; i < n; i++) {
            for (int j = i+1; j < n; j++) {
                for (int k = j+1; k < n; k++) {
                    int sum = number[i] + number[j] + number[k];

                    if (sum == 0) {
                        answer += 1;
                    }
                }
            }
        }

        return answer;
    }
}
```
