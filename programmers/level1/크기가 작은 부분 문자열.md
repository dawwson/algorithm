> 문제 유형 : 연습 문제 (문자열)  
> 문제 제목 : 크기가 작은 부분 문자열  
> 문제 링크 : https://school.programmers.co.kr/learn/courses/30/lessons/147355

<br>

### Intuition

---

- `t`에서 길이가 `p`와 같은 부분 문자열을 모두 확인해야 한다.
- 비교 대상과 `p`의 길이가 항상 같으므로, 숫자로 변환하지 않고 문자열 비교로도 크기를 판단할 수 있다.
  - 길이가 같은 숫자 문자열에서는 사전순 비교 결과와 숫자 크기 비교 결과가 같다.
- `p`의 길이만큼 잘라낸 부분 문자열이 `p`보다 작거나 같으면 정답에 포함한다.
- `p`의 길이는 최대 `18`이므로 숫자로 변환한다면 `long` 범위 안에서도 처리할 수 있다.

<br>

### Approach

---

- 반복 횟수는 `t.length() - p.length() + 1`이다.
  - `p`와 같은 길이의 부분 문자열을 만들 수 있는 시작 위치의 개수이다.
- `i = 0`부터 가능한 마지막 시작 위치까지 순회한다.
- `t.substring(i, i + p.length())`로 길이가 `p`와 같은 부분 문자열 `sub`를 만든다.
- `sub.compareTo(p) < 1`이면 `sub`가 `p`보다 작거나 같다는 뜻이므로 `answer`를 증가시킨다.
- 모든 부분 문자열을 확인한 뒤 `answer`를 반환한다.

<br>

### Complexity

---

- Time complexity: `O(n * m)`
  - `n`은 `t`의 길이, `m`은 `p`의 길이이다.
  - 시작 위치는 `n - m + 1`개이다.
  - 각 위치마다 길이 `m`의 부분 문자열을 만들고 비교하므로 전체 시간 복잡도는 `O(n * m)`으로 표현할 수 있다.

- Space complexity: `O(m)`
  - 매 반복마다 길이 `m`의 부분 문자열 `sub`를 생성한다.

<br>

### Code (Java)

---

```java
class Solution {
    public int solution(String t, String p) {
        int MAX_DIGIT = 9;
        int answer = 0;
        int n = t.length() - p.length() + 1; // 반복횟수

        for (int i = 0; i < n; i++) {
            String sub = t.substring(i, i + p.length());

            boolean isSmaller = sub.compareTo(p) < 1;

            if (isSmaller) {
                answer += 1;
            }
        }

        return answer;
    }
}
```

### Better Code (Java)

---

#### 슬라이딩 윈도우

```java
class Solution {
    public int solution(String t, String p) {
        int answer = 0;
        int n = t.length();
        int m = p.length();

        // p는 최대 18자리이므로 long으로 변환해 비교할 수 있다.
        long target = Long.parseLong(p);

        // 현재 확인 중인 길이 m의 부분 숫자
        long window = 0;

        // 가장 왼쪽 자리 숫자를 제거할 때 사용할 10^(m - 1)
        long pow = 1;

        // 첫 번째 길이 m 구간의 숫자 값을 만든다.
        for (int i = 0; i < m; i++) {
            window = window * 10 + (t.charAt(i) - '0');

            if (i < m - 1) {
                pow *= 10;
            }
        }

        // 첫 번째 구간도 비교 대상에 포함한다.
        if (window <= target) {
            answer++;
        }

        // 윈도우를 한 칸씩 오른쪽으로 이동하며 다음 부분 숫자를 만든다.
        for (int i = m; i < n; i++) {
            // 이전 구간의 가장 왼쪽 숫자
            int left = t.charAt(i - m) - '0';

            // 새로 추가할 오른쪽 숫자
            int right = t.charAt(i) - '0';

            // 왼쪽 숫자를 제거하고, 한 자리 올린 뒤, 오른쪽 숫자를 붙인다.
            window = (window - left * pow) * 10 + right;

            if (window <= target) {
                answer++;
            }
        }

        return answer;
    }
}
```

- `p`의 길이가 최대 `18`이므로 `long` 범위 안에서 숫자로 변환할 수 있다.
- 처음 `m`자리 숫자를 `window`에 저장한다.
- 이후 한 칸씩 이동할 때 가장 왼쪽 숫자를 제거하고, 오른쪽 숫자 하나만 추가한다.
- 매번 `substring()`을 만들거나 길이 `m` 전체를 다시 비교하지 않는다.
- 시간 복잡도는 `O(n)`, 공간 복잡도는 `O(1)`로 개선된다.
