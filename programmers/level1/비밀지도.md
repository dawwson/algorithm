### Intuition

---

- 각 배열의 정수는 한 행을 이진수로 표현한 값이며, 이진수의 각 자리가 지도의 벽과 공백을 나타낸다.
- 두 지도 중 하나라도 벽인 위치는 최종 지도에서도 벽이다.
  - 이 규칙은 두 비트 중 하나라도 `1`이면 `1`이 되는 비트 OR 연산과 같다.
- 따라서 같은 행의 `arr1[i]`와 `arr2[i]`를 `|` 연산한 뒤, 결과의 각 비트를 `#` 또는 공백으로 변환하면 된다.
- 이진수 문자열은 앞쪽의 `0`이 생략되므로, 항상 길이가 `n`이 되도록 앞을 `0`으로 채워야 한다.

<br>

### Approach

---

- 길이가 `n`인 정답 배열 `answer`를 생성한다.
- 각 행 `i`에 대해 `arr1[i] | arr2[i]`를 계산해 두 지도를 합친다.
- `Integer.toBinaryString()`으로 OR 연산 결과를 이진수 문자열로 변환한다.
- 이진수 문자열을 `long`으로 변환한 뒤 `String.format()`을 사용해 앞을 `0`으로 채우고 길이를 `n`으로 맞춘다.
  - 여기서 `long` 값은 이진수의 수치가 아니라 `0`과 `1`로 이루어진 문자열을 10진수 숫자처럼 읽은 값이며, 자릿수를 맞추기 위한 용도로만 사용한다.
- 길이 `n`의 문자열을 순회한다.
  - 문자가 `1`이면 벽이므로 `#`을 추가한다.
  - 문자가 `0`이면 공백을 추가한다.
- 완성한 행을 `answer[i]`에 저장하고, 모든 행을 처리한 뒤 반환한다.

<br>

### Complexity

---

- Time complexity: `O(n^3)`
  - 총 `n^2`개의 칸을 확인한다.
  - 현재 코드는 불변 객체인 `String`에 `row += ...`로 문자를 반복해서 붙인다. 한 행을 만들 때 누적 문자열 복사가 `O(n^2)`, 전체 `n`개 행에서는 `O(n^3)`이 된다.
  - 문제에서 `n`은 최대 16으로 작지만, `StringBuilder`를 사용하면 전체 시간 복잡도를 `O(n^2)`으로 줄일 수 있다.

- Space complexity: `O(n^2)`
  - 반환하는 `n`개의 문자열이 각각 길이 `n`이므로 정답 배열이 `O(n^2)` 공간을 차지한다.
  - 정답을 제외한 행별 임시 문자열은 `O(n)` 공간을 사용한다.

<br>

### Code (Java)

---

```java
class Solution {
    public String[] solution(int n, int[] arr1, int[] arr2) {

        String[] answer = new String[n];

        for (int i = 0; i < n; i++) {
            int num1 = arr1[i];
            int num2 = arr2[i];

            String binary = Integer.toBinaryString(num1 | num2);
            String decrypted = String.format("%0"+n+"d", Long.parseLong(binary));

            String row = "";

            for (int j = 0; j < n; j++) {
                boolean wall = decrypted.charAt(j) == '1';

                if (wall) {
                    row += "#";
                } else {
                    row += " ";
                }
            }

            answer[i] = row;
        }

        return answer;
    }
}
```

### Better Code (Java)

---

```java
class Solution {
    public String[] solution(int n, int[] arr1, int[] arr2) {
        String[] answer = new String[n];

        for (int i = 0; i < n; i++) {
            int merged = arr1[i] | arr2[i];
            StringBuilder row = new StringBuilder(n);

            for (int bit = n - 1; bit >= 0; bit--) {
                if ((merged & (1 << bit)) != 0) {
                    row.append('#');
                } else {
                    row.append(' ');
                }
            }

            answer[i] = row.toString();
        }

        return answer;
    }
}
```

- OR 연산 결과의 비트를 가장 높은 자리부터 직접 확인하므로 이진수 문자열 변환과 `0` 채우기가 필요 없다.
- `merged & (1 << bit)`가 `0`이 아니면 해당 위치의 비트가 `1`이므로 `#`을 추가한다.
  - `1 << bit`은 숫자 `1`을 왼쪽으로 `bit`칸 옮겨 검사할 위치만 1로 만드는 연산이다.

    ```
    1 << 0 → 00001
    1 << 1 → 00010
    1 << 2 → 00100
    1 << 3 → 01000
    1 << 4 → 10000
    ```

    예를 들어 `merged = 10101`이고 왼쪽에서 두 번째 비트를 확인하려면 다음과 같다.

    ```
    merged     10101
    mask       01000  // 1 << 3
               -----
    AND 결과    00000
    ```

    `AND` 연산 `&`은 양쪽 비트가 모두 `1`일 때만 `1`이 된다. 결과가 `0`이므로 해당 위치는 공백이다.

  - 즉, `merged & (1 << bit)) != 0` 조건문은 'merged의 bit번째 자리가 1인가?'라는 뜻이다.

- `StringBuilder`로 각 행을 만들어 반복적인 문자열 복사를 피한다.
- 시간 복잡도는 출력할 모든 칸을 한 번씩 확인하므로 `O(n^2)`이다.
- 공간 복잡도는 반환값을 포함하면 `O(n^2)`, 반환값을 제외한 추가 공간은 행을 만드는 `O(n)`이다.

<br>

### Code (Python)

---

```python
def solution(n, arr1, arr2):
    answer = []

    for i in range(n):
        decrypted = ''

        bin_format = '0' + str(n) +'b'  # 0b를 떼고 앞에 0을 붙여서 n자리로 맞춘다
        result = format(arr1[i] | arr2[i], bin_format)
        print(bin_format)

        for j in range(n):
            if result[j] == '1':
                decrypted = decrypted + '#'
            else:
                decrypted = decrypted + ' '

        answer.append(decrypted)
    return answer
```

- 같은 행의 두 정수를 비트 OR 연산해 두 지도를 합친다.
- `format(..., '0' + str(n) + 'b')`로 OR 연산 결과를 앞이 `0`으로 채워진 `n`자리 이진수 문자열로 만든다.
- 각 문자가 `1`이면 `#`, `0`이면 공백으로 바꾸어 한 행을 완성한다.
- 완성된 행을 `answer`에 추가하고 모든 행을 처리한 뒤 반환한다.
- 코드의 `print(bin_format)`은 풀이 결과와 관계없는 디버깅 출력이다.
- 문자열을 `+`로 반복 결합하므로 일반화한 시간 복잡도는 `O(n^3)`, 공간 복잡도는 반환값을 포함해 `O(n^2)`이다.
