### Intuition

---

- 로또 번호 중 `0`은 알아볼 수 없는 번호이므로, 당첨 번호와 일치할 수도 있고 일치하지 않을 수도 있다.
- 따라서 현재 확실히 맞힌 번호 개수와 알 수 없는 번호 개수를 따로 세면 된다.
- 최저 순위는 `0`이 모두 틀렸다고 가정했을 때의 순위이다.
- 최고 순위는 `0`이 모두 당첨 번호라고 가정했을 때의 순위이다.
- 맞힌 개수와 순위의 관계는 고정되어 있으므로 `Map`이나 함수로 변환하면 된다.

<br>

### Approach

---

- `rank` Map에 맞힌 개수별 순위를 저장한다.
  - 6개 일치하면 1등, 5개 일치하면 2등이다.
  - 1개 이하로 맞히면 모두 6등이므로 `0`, `1` 모두 6등으로 처리한다.
- `matched`에는 확실히 당첨 번호와 일치한 개수를 저장한다.
- `unknown`에는 `0`의 개수를 저장한다.
- `lottos`를 순회한다.
  - 현재 번호가 `0`이면 `unknown`을 증가시킨다.
  - `0`이 아니면 `win_nums` 안에 같은 번호가 있는지 확인하고, 있으면 `matched`를 증가시킨다.
- 최고 순위는 `matched + unknown`개가 맞았다고 보고 구한다.
- 최저 순위는 `matched`개만 맞았다고 보고 구한다.
- `[최고 순위, 최저 순위]`를 반환한다.

<br>

### Complexity

---

- Time complexity: `O(n * m)`
  - `n`은 `lottos`의 길이, `m`은 `win_nums`의 길이이다.
  - 현재 Java 코드는 각 로또 번호마다 `win_nums`를 스트림으로 다시 확인한다.
  - 실제 문제에서는 두 배열 길이가 항상 6으로 고정되어 있어 상수 시간으로 볼 수도 있다.

- Space complexity: `O(1)`
  - 입력 크기와 무관하게 `rank`, `matched`, `unknown`만 사용한다.

<br>

### Code (Java)

---

```java
import java.util.Map;
import java.util.stream.IntStream;

class Solution {
    public int[] solution(int[] lottos, int[] win_nums) {
        // 일치 개수 : 순위
        Map<Integer, Integer> rank =  Map.of(
            6, 1,
            5, 2,
            4, 3,
            3, 4,
            2, 5,
            1, 6,
            0, 6
        );
        int matched = 0;
        int unknown = 0;

        for (int i = 0; i < lottos.length; i++) {
            int num = lottos[i];

            if (num == 0) {
                unknown += 1;
                continue;
            }

            boolean isMatched = IntStream.of(win_nums).anyMatch(win_num -> win_num == num);
            if (isMatched) {
                matched += 1;
            }
        }

        int maxRank = rank.get(matched + unknown);
        int minRank = rank.get(matched);

        return new int[]{maxRank, minRank};
    }
}
```

### Better Code (Java)

---

```java
class Solution {
    public int[] solution(int[] lottos, int[] win_nums) {
        boolean[] winning = new boolean[46];

        for (int num : win_nums) {
            winning[num] = true;
        }

        int matched = 0;
        int unknown = 0;

        for (int num : lottos) {
            if (num == 0) {
                unknown++;
                continue;
            }

            if (winning[num]) {
                matched++;
            }
        }

        return new int[]{getRank(matched + unknown), getRank(matched)};
    }

    private int getRank(int matched) {
        if (matched <= 1) {
            return 6;
        }

        return 7 - matched;
    }
}
```

- `win_nums`를 매번 순회하지 않고, 당첨 번호 여부를 `boolean` 배열에 미리 표시한다.
- 로또 번호의 범위가 `1`부터 `45`까지로 작고 고정되어 있으므로 배열 인덱스로 빠르게 확인할 수 있다.
- 맞힌 개수와 순위의 관계도 `Map` 대신 `getRank()` 함수로 계산해 고정 데이터를 줄였다.
- 시간 복잡도는 일반화하면 `O(n + m)`, 문제 조건처럼 번호 개수가 고정이면 `O(1)`이다.
- 공간 복잡도는 번호 범위가 고정되어 있으므로 `O(1)`이다.

### Code (Python)

---

```python
def solution(lottos, win_nums):
    win_set = set(win_nums)
    matched = 0
    unknown = 0

    for num in lottos:
        if num == 0:
            unknown += 1
            continue

        if num in win_set:
            matched += 1

    return [get_rank(matched + unknown), get_rank(matched)]


def get_rank(matched):
    if matched <= 1:
        return 6

    return 7 - matched
```
