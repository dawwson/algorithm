> 풀이 날짜 : 2026.07.01  
> 문제 유형 : 2023 KAKAO BLIND RECRUITMENT (구현 / 문자열 / 날짜 계산)  
> 문제 제목 : 개인정보 수집 유효기간  
> 문제 링크 : https://school.programmers.co.kr/learn/courses/30/lessons/150370

<br>

### Intuition

---

- 모든 달은 `28`일까지 있다고 가정하므로 날짜 계산을 실제 달력 기준으로 처리할 필요가 없다.
- `YYYY.MM.DD` 형식의 날짜를 `년 * 12 * 28 + 월 * 28 + 일` 형태의 총 일수로 바꾸면 두 날짜의 차이를 쉽게 계산할 수 있다.
- 개인정보 수집일과 오늘 날짜의 차이가 약관 유효기간보다 크거나 같으면 이미 파기해야 하는 개인정보이다.
  - 예를 들어 유효기간이 `12`개월이면 `12 * 28`일이 지난 시점부터 만료된 것으로 판단할 수 있다.
- 약관 종류별 유효기간을 빠르게 조회하기 위해 `Map`에 저장한다.

<br>

### Approach

---

- `terms`를 순회하며 약관 종류와 유효기간을 분리한다.
  - key는 약관 종류, value는 유효기간 개월 수이다.
  - 이후 개인정보를 검사할 때 약관 종류만으로 유효기간을 바로 찾기 위해 `HashMap`을 사용한다.
- `privacies`를 순서대로 순회한다.
  - 각 개인정보에서 수집 날짜와 약관 종류를 분리한다.
  - 약관 종류에 해당하는 유효기간을 `termsMap`에서 조회한다.
  - 수집 날짜와 오늘 날짜를 각각 총 일수로 환산한 뒤 차이를 계산한다.
- 날짜 차이가 `유효기간 * 28` 이상이면 유효기간이 지난 것이므로 현재 개인정보 번호를 정답 리스트에 추가한다.
  - 문제의 개인정보 번호는 `1`번부터 시작하므로 배열 인덱스 `i`에 `1`을 더해 저장한다.
- 모든 개인정보를 확인한 뒤 `List<Integer>`를 `int[]`로 변환해 반환한다.

<br>

### Complexity

---

- Time complexity: `O(t + p)`
  - `t`는 `terms`의 길이, `p`는 `privacies`의 길이이다.
  - 약관 정보를 Map에 저장하기 위해 `terms`를 한 번 순회한다.
  - 각 개인정보도 한 번씩만 확인하므로 전체 시간 복잡도는 `O(t + p)`이다.

- Space complexity: `O(t + p)`
  - 약관 종류별 유효기간을 저장하는 `Map`에 최대 `t`개의 항목이 들어간다.
  - 정답 리스트에는 최악의 경우 모든 개인정보 번호가 들어갈 수 있으므로 `O(p)` 공간이 필요하다.

<br>

### Code (Java)

---

```java
import java.util.*;
import java.time.LocalDate;
import java.time.temporal.ChronoUnit;

class Solution {

    private int MAX_DAY = 28;

    public int[] solution(String today, String[] terms, String[] privacies) {

        List<Integer> answers = new ArrayList<>();

        // {약관종류 : 유효기간} Map 구성
        Map<String, Integer> termsMap = new HashMap<>();

        for (String term : terms) {
            String[] splitted = term.split(" ");

            String termType = splitted[0];
            int period = Integer.parseInt(splitted[1]);

            termsMap.put(termType, period);
        }

        // 개인정보 순회
        for (int i = 0; i < privacies.length; i++) {
            int number = i + 1;

            String[] parts = privacies[i].split(" ");

            String privacyDates = parts[0]; // format: YYYY.MM.DD
            String termType = parts[1];

            int expirationPeriod = termsMap.get(termType);
            int diffDays = getDiffDays(privacyDates, today);

            boolean isExpired = diffDays >= expirationPeriod * MAX_DAY;

            if (isExpired) {
                answers.add(number);
            }
        }

        return answers.stream()
            .mapToInt(Integer::intValue)
            .toArray();
    }

    // YYYY.MM.DD 포맷으로 날짜 차이 계산
    private int getDiffDays(String startDate, String endDate) {

        String[] startDates = startDate.split("\\.");
        String[] endDates = endDate.split("\\.");

        int startYear = Integer.parseInt(startDates[0]);
        int startMonth = Integer.parseInt(startDates[1]);
        int startDay = Integer.parseInt(startDates[2]);

        int endYear = Integer.parseInt(endDates[0]);
        int endMonth = Integer.parseInt(endDates[1]);
        int endDay = Integer.parseInt(endDates[2]);

        // 일수로 환산
        int startDays = startYear * 12 * MAX_DAY + startMonth * MAX_DAY + startDay;
        int endDays = endYear * 12 * MAX_DAY + endMonth * MAX_DAY + endDay;
        int diffDays = endDays - startDays;

        return diffDays;
    }
}
```

### Better Code (Java)

---

```java
import java.util.*;

class Solution {

    private static final int MAX_DAY = 28;

    public int[] solution(String today, String[] terms, String[] privacies) {
        int[] periods = new int[26];

        for (String term : terms) {
            char type = term.charAt(0);
            int period = Integer.parseInt(term.substring(2));

            periods[type - 'A'] = period;
        }

        int todayDays = toDays(today);
        int[] answer = new int[privacies.length];
        int count = 0;

        for (int i = 0; i < privacies.length; i++) {
            String privacy = privacies[i];
            String date = privacy.substring(0, 10);
            char type = privacy.charAt(11);

            int collectedDays = toDays(date);
            int expirationDays = periods[type - 'A'] * MAX_DAY;

            if (todayDays - collectedDays >= expirationDays) {
                answer[count++] = i + 1;
            }
        }

        return Arrays.copyOf(answer, count);
    }

    private int toDays(String date) {
        int year = Integer.parseInt(date.substring(0, 4));
        int month = Integer.parseInt(date.substring(5, 7));
        int day = Integer.parseInt(date.substring(8, 10));

        return year * 12 * MAX_DAY + month * MAX_DAY + day;
    }
}
```

- 시간 복잡도는 기존 풀이와 같은 `O(t + p)`이다.
  - 모든 약관과 개인정보를 최소 한 번은 확인해야 하므로 점근적으로 더 줄이기는 어렵다.
- 약관 종류가 알파벳 대문자 1글자라는 조건을 이용해 `HashMap` 대신 길이 `26`의 배열을 사용한다.
- `split()`으로 매번 문자열 배열을 만들지 않고, 고정된 위치를 `substring()`과 `charAt()`으로 읽는다.
- `List<Integer>`와 `stream()` 변환을 사용하지 않아 박싱, 언박싱 비용을 줄일 수 있다.
- 정답 배열을 제외한 추가 공간은 `O(1)`로 줄어든다.
