### Intuition

---

- 성격 유형은 `RT`, `CF`, `JM`, `AN`의 네 지표로 이루어지며, 각 지표에서 점수가 더 높은 문자를 선택해야 한다.
- 각 질문의 `survey[i]`에서 첫 번째 문자는 비동의 선택지, 두 번째 문자는 동의 선택지로 점수를 얻는다.
- 선택값 `4`는 모르겠음을 의미하므로 어느 성격 유형에도 점수를 주지 않는다.
- 선택값이 `1`부터 `3`이면 첫 번째 문자에 `4 - choice`점을, `5`부터 `7`이면 두 번째 문자에 `choice - 4`점을 더할 수 있다.
- 같은 지표의 두 유형이 동점이면 사전순으로 빠른 유형을 선택해야 한다.
  - 각 지표가 `RT`, `CF`, `JM`, `AN`처럼 사전순으로 주어지므로, 점수가 같을 때 첫 번째 문자를 선택하면 된다.
- 유형별 누적 점수를 빠르게 갱신하고 조회하기 위해 `HashMap`을 사용한다.

<br>

### Approach

---

- 8개의 성격 유형 문자를 `types` Map에 넣고 점수를 모두 `0`으로 초기화한다.
- 모든 선택 결과를 순회한다.
  - `survey[i]`의 첫 번째 문자를 `negative`, 두 번째 문자를 `positive`로 가져온다.
  - 선택값이 `1`부터 `3`이면 `negative`에 `4 - choice`점을 더한다.
  - 선택값이 `5`부터 `7`이면 `positive`에 `choice - 4`점을 더한다.
  - 선택값이 `4`이면 점수를 더하지 않는다.
- `R`과 `T`, `C`와 `F`, `J`와 `M`, `A`와 `N`의 누적 점수를 차례대로 비교한다.
- 두 번째 유형의 점수가 더 높을 때만 두 번째 문자를 선택하고, 그 외에는 첫 번째 문자를 선택한다.
  - 따라서 동점인 경우에도 사전순으로 빠른 유형이 선택된다.
- 선택한 네 문자를 이어 붙여 최종 성격 유형을 반환한다.

<br>

### Complexity

---

- Time complexity: `O(n)`
  - `n`은 `survey`와 `choices`의 길이이다.
  - 모든 질문을 한 번 순회하고, 마지막 네 지표 비교는 상수 시간에 수행한다.

- Space complexity: `O(1)`
  - 성격 유형은 항상 8개이므로 Map에 저장되는 항목 수가 입력 크기와 관계없이 일정하다.

<br>

### Code (Java)

---

```java
import java.util.Map;
import java.util.HashMap;

class Solution {
    public String solution(String[] survey, int[] choices) {

        String answer = "";

        Map<Character, Integer> types = new HashMap<>();

        char[] basicTypes = {'R', 'T', 'C', 'F', 'J', 'M', 'A', 'N'};

        for (char t : basicTypes) {
            types.put(t, 0);
        }

        for (int i = 0; i < choices.length; i++) {
            char negative = survey[i].charAt(0);
            char positive = survey[i].charAt(1);
            int choice = choices[i];

            if (choice >= 1 && choice <= 3) {
                int score = 4 - choice; // 1->3점, 2->2점, 3->1점
                types.put(negative, types.get(negative) + score);
            } else if (choice >= 5 && choice <= 7) {
                int score = choice - 4; // 5->1점, 6->2점, 7->3점
                types.put(positive, types.get(positive) + score);
            }
        }

        if (types.get('R') < types.get('T')) {
            answer += 'T';
        } else {
            answer += 'R';
        }
        if (types.get('C') < types.get('F')) {
            answer += 'F';
        } else {
            answer += 'C';
        }
        if (types.get('J') < types.get('M')) {
            answer += 'M';
        } else {
            answer += 'J';
        }
        if (types.get('A') < types.get('N')) {
            answer += 'N';
        } else {
            answer += 'A';
        }

        return answer;
    }
}
```

### Better Code (Java)

```java
class Solution {
    public String solution(String[] survey, int[] choices) {
        int[] scores = new int[26];

        for (int i = 0; i < survey.length; i++) {
            char first = survey[i].charAt(0);
            char second = survey[i].charAt(1);
            int choice = choices[i];

            if (choice < 4) {
                scores[first - 'A'] += 4 - choice;
            } else if (choice > 4) {
                scores[second - 'A'] += choice - 4;
            }
        }

        String[] indicators = {"RT", "CF", "JM", "AN"};
        StringBuilder answer = new StringBuilder();

        for (String indicator : indicators) {
            char first = indicator.charAt(0);
            char second = indicator.charAt(1);

            if (scores[first - 'A'] >= scores[second - 'A']) {
                answer.append(first);
            } else {
                answer.append(second);
            }
        }

        return answer.toString();
    }
}
```

- 성격 유형 문자를 알파벳 인덱스로 바꾸어 `int[]`에 점수를 저장하므로 Map 초기화와 조회가 필요 없다.
- `choice`와 중립값 `4`의 차이로 점수를 계산해 선택값별 점수표를 별도로 만들지 않는다.
- 네 지표를 배열에 저장하고 반복문으로 비교해 같은 형태의 조건문을 줄였다.
- `StringBuilder`를 사용해 선택된 유형을 이어 붙인다.
- 시간 복잡도는 `O(n)`, 공간 복잡도는 `O(1)`로 기존 코드와 동일하다.

### Code (Python)

---

```python
def solution(survey, choices):
    answer = ''
    types = {
        'R': 0, 'T': 0,
        'C': 0, 'F': 0,
        'J': 0, 'M': 0,
        'A': 0, 'N': 0
    }

    for index, choice in enumerate(choices):
        type1, type2 = list(survey[index])
        if choice == 1:
            types[type1] += 3
        if choice == 2:
            types[type1] += 2
        if choice == 3:
            types[type1] += 1
        if choice == 5:
            types[type2] += 1
        if choice == 6:
            types[type2] += 2
        if choice == 7:
            types[type2] += 3

    if types['R'] < types['T']:
        answer += 'T'
    else:
        answer += 'R'

    if types['C'] < types['F']:
        answer += 'F'
    else:
        answer += 'C'

    if types['J'] < types['M']:
        answer += 'M'
    else:
        answer += 'J'

    if types['A'] < types['N']:
        answer += 'N'
    else:
        answer += 'A'

    return answer
```

- 8개 성격 유형의 점수를 딕셔너리에 `0`으로 초기화한다.
- 각 질문의 선택값에 따라 `survey`의 첫 번째 또는 두 번째 유형에 `1`점부터 `3`점까지 더한다.
- 선택값이 `4`이면 어느 유형에도 점수를 더하지 않는다.
- 네 지표의 점수를 차례대로 비교하며 더 높은 유형을 정답에 추가한다.
- 점수가 같으면 각 지표에서 사전순으로 앞선 첫 번째 유형을 선택한다.
- 시간 복잡도는 `O(n)`, 공간 복잡도는 `O(1)`이다.
