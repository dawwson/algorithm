### Intuition

---

- 벽의 일부 구역만 다시 칠해야 하고, 롤러 한 번으로 연속된 `m`칸을 칠할 수 있다.
- `section`은 다시 칠해야 하는 구역 번호가 오름차순으로 주어진다.
- 한 번 칠할 때 가능한 한 현재 필요한 구역부터 오른쪽으로 멀리 칠하는 것이 유리하다.
  - 이미 이전에 칠한 범위 안에 들어온 구역은 따로 처리할 필요가 없다.
  - 따라서 아직 칠해지지 않은 가장 왼쪽 구역을 만날 때마다 그 위치부터 롤러를 한 번 사용하면 된다.
- `section`이 오름차순으로 정렬되어 있으므로, 매번 현재 칠해야 하는 가장 왼쪽 구역부터 최대한 오른쪽까지 칠하는 선택이 최선이므로 Greedy로 풀 수 있다.

<br>

### Approach

---

- `painted`에 현재까지 칠해진 마지막 구역 번호를 저장한다.
- `section`을 왼쪽부터 순회한다.
- 현재 구역 `position`이 `painted` 이하라면 이미 칠해진 구역이므로 넘어간다.
- 현재 구역이 아직 칠해지지 않았다면 롤러를 한 번 사용한다.
- 이때 `position`부터 `position + m - 1`까지 칠할 수 있으므로 `painted`를 그 값으로 갱신한다.
- 모든 다시 칠해야 하는 구역을 확인한 뒤 롤러 사용 횟수 `answer`를 반환한다.

<br>

### Complexity

---

- Time complexity: `O(k)`
  - `k`는 `section`의 길이이다.
  - 다시 칠해야 하는 구역만 한 번씩 순회한다.

- Space complexity: `O(1)`
  - 입력 배열 외에 `answer`, `painted`만 사용한다.

<br>

### Code (Java)

---

```java
class Solution {
    public int solution(int n, int m, int[] section) {
        int answer = 0;
        int painted = 0;

        for (int position : section) {
            if (position <= painted) {
                continue;
            }

            answer += 1;
            painted = position + m - 1;
        }

        return answer;
    }
}
```

<br>

### Better Code (Java)

---

```java
class Solution {
    public int solution(int n, int m, int[] section) {
        int count = 0;
        int coveredUntil = 0;

        for (int start : section) {
            if (start > coveredUntil) {
                count++;
                coveredUntil = start + m - 1;
            }
        }

        return count;
    }
}
```

- 기존 Java 코드와 같은 그리디 방식이며 시간 복잡도와 공간 복잡도도 동일하다.
- `coveredUntil`이라는 변수명으로 현재까지 칠해진 마지막 위치라는 의미를 조금 더 직접적으로 드러냈다.
- `continue` 없이 아직 칠하지 않은 구역일 때만 처리하도록 작성해 흐름을 더 짧게 만들 수 있다.

<br>

### Code (Python)

---

```python
def solution(n, m, section):
    paint_count = 1
    prev = section[0]

    for s in section:
        if s - prev >= m:
            paint_count += 1
            prev = s

    return paint_count
```

- Python 코드는 현재 롤러를 새로 시작한 위치를 `prev`에 저장하는 방식이다.
- 다음 구역 `s`가 `prev`로부터 `m`칸 이상 떨어져 있으면 기존 롤러 범위로는 칠할 수 없으므로 롤러 횟수를 늘리고 `prev`를 갱신한다.
- Java 코드가 `마지막으로 칠해진 위치`를 기준으로 판단한다면, Python 코드는 `마지막으로 롤러를 시작한 위치`를 기준으로 판단하는 차이가 있다.
- 두 방식 모두 다시 칠해야 하는 구역을 왼쪽부터 보며 필요한 순간에만 롤러를 추가하는 같은 그리디 풀이이다.
