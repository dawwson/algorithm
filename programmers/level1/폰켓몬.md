### Intuition

---

- `nums`에는 폰켓몬의 종류 번호가 들어 있으며, 전체 `N`마리 중 정확히 `N / 2`마리를 선택해야 한다.
- 같은 종류를 여러 마리 선택해도 종류의 수는 늘어나지 않으므로, 먼저 서로 다른 폰켓몬 종류의 개수를 알아야 한다.
- 선택할 수 있는 종류의 수는 다음 두 값에 의해 제한된다.
  - 보유한 서로 다른 폰켓몬 종류의 수
  - 선택할 수 있는 폰켓몬 수인 `N / 2`
- 따라서 두 값 중 작은 값이 선택할 수 있는 폰켓몬 종류의 최댓값이다.
- 중복을 제거하면서 서로 다른 종류의 수를 세기 위해 `HashSet`을 사용한다.

<br>

### Approach

---

- 빈 `HashSet<Integer>`를 생성한다.
- `nums`를 순회하며 각 폰켓몬의 종류 번호를 Set에 추가한다.
  - Set에는 중복된 값이 저장되지 않으므로, 순회가 끝난 뒤 `set.size()`는 서로 다른 종류의 수가 된다.
- 선택해야 하는 폰켓몬 수 `nums.length / 2`를 `selected`에 저장한다.
- `selected`가 서로 다른 종류의 수보다 크거나 같으면 모든 종류를 하나씩 선택할 수 있으므로 `set.size()`를 반환한다.
- 그렇지 않으면 선택 가능한 마릿수가 종류의 수를 제한하므로 `selected`를 반환한다.

<br>

### Complexity

---

- Time complexity: `O(n)`
  - `n`은 `nums`의 길이이다.
  - 모든 원소를 한 번씩 순회하며 `HashSet`에 추가한다. 해시 삽입은 평균적으로 `O(1)`이다.

- Space complexity: `O(n)`
  - 최악의 경우 모든 폰켓몬이 서로 다른 종류이면 Set에 `n`개의 값이 저장된다.

<br>

### Code (Java)

---

```java
import java.util.*;

class Solution {
    public int solution(int[] nums) {

        Set<Integer> set = new HashSet<>();

        for (int num : nums) {
            set.add(num);
        }

        int selected = nums.length / 2;

        if (selected >= set.size()) {
            return set.size();
        }

        return selected;
    }
}
```

### Better Code (Java)

---

```java
import java.util.HashSet;
import java.util.Set;

class Solution {
    public int solution(int[] nums) {
        Set<Integer> types = new HashSet<>();

        for (int num : nums) {
            types.add(num);
        }

        return Math.min(nums.length / 2, types.size());
    }
}
```

- 기존 코드의 조건문은 `Math.min()`으로 표현할 수 있다.
- 필요한 클래스만 명시적으로 import하고, `types`라는 이름으로 폰켓몬 종류를 저장한다는 의미를 드러냈다.
- 알고리즘과 시간 및 공간 복잡도는 기존 코드와 동일하다.

### Code (Python)

---

```python
def solution(nums):
    type_set = set(nums)
    select_count = len(nums) / 2

    if select_count >= len(type_set):
        return len(type_set)
    return select_count
```
