### Intuition

---

- 각 기사 번호의 약수 개수가 해당 기사가 사용할 무기의 공격력이다.
- 공격력이 `limit`보다 크면 실제로는 `power`만큼의 공격력을 가진 무기를 구매한다.
- 따라서 `1`부터 `number`까지 각 수의 약수 개수를 구하고, 제한을 넘는지만 확인하면 된다.
- 단순히 모든 수 `n`에 대해 `1`부터 `n / 2`까지 확인하면 약수 개수는 구할 수 있지만, 불필요한 반복이 많다.
- 약수는 짝을 이루므로 제곱근까지만 확인하거나, 전체 숫자의 약수 개수를 배수 단위로 미리 누적하면 더 효율적으로 풀 수 있다.

<br>

### Approach

---

- `1`부터 `number`까지 순회하며 각 번호의 약수 개수를 구한다.
- `getFactorsCount(n)`은 `n`의 약수 개수를 반환한다.
- 현재 Java 코드는 `1`을 미리 포함한 뒤, `1`부터 `n / 2`까지 나누어떨어지는 수를 세고 마지막에 자기 자신을 포함하는 방식이다.
- 구한 약수 개수가 `limit`보다 크면 `power`를 더한다.
- `limit` 이하라면 약수 개수를 그대로 더한다.
- 모든 기사 번호를 처리한 뒤 필요한 철의 무게 합계를 반환한다.

<br>

### Complexity

---

- Time complexity: `O(number^2)`
  - 각 수 `n`마다 `1`부터 `n / 2`까지 확인한다.
  - 전체 반복 수는 대략 `1 + 2 + ... + number / 2` 형태이므로 `O(number^2)`이다.

- Space complexity: `O(1)`
  - 정답과 현재 약수 개수 외에 입력 크기에 비례하는 추가 자료구조를 사용하지 않는다.

<br>

### Code (Java)

---

```java
class Solution {
    public int solution(int number, int limit, int power) {
        int answer = 0;

        for (int i = 1; i <= number; i++) {
            int factorsCount = getFactorsCount(i);

            if (factorsCount > limit) {
                answer += power;
            } else {
                answer += factorsCount;
            }
        }

        return answer;
    }

    private int getFactorsCount(int n) {
        // 1은 언제나 포함
        int count = 1;

        for (int i = 1; i <= n/2; i++) {
            if (n % i == 0) {
                count += 1;
            }
        }

        return count;
    }
}
```

### Better Code (Java)

---

#### 약수 개수 미리 구하기

```java
class Solution {
    public int solution(int number, int limit, int power) {
        int[] divisorCounts = new int[number + 1];

        // i는 약수 후보이다.
        // i의 배수들은 모두 i를 약수로 가지므로 개수를 1씩 증가시킨다.
        for (int i = 1; i <= number; i++) {
            for (int j = i; j <= number; j += i) {
                divisorCounts[j]++;
            }
        }

        int answer = 0;

        for (int i = 1; i <= number; i++) {
            if (divisorCounts[i] > limit) {
                answer += power;
            } else {
                answer += divisorCounts[i];
            }
        }

        return answer;
    }
}
```

- 기존 Java 코드는 각 숫자마다 `n / 2`까지 직접 확인하므로 반복이 많다.
- 개선 코드는 `i`를 약수로 가지는 숫자들, 즉 `i`, `2i`, `3i` ... 의 약수 개수를 한 번씩 증가시킨다.
- 이렇게 하면 `1`부터 `number`까지의 약수 개수를 한꺼번에 계산할 수 있다.
- 시간 복잡도는 `O(number log number)`, 공간 복잡도는 `O(number)`이다.

<br>

### Code (Python)

---

```python
def solution(number, limit, power):
    # 1번 기사의 약수는 1개이므로 미리 더해 둔다.
    weight = 1

    for n in range(2, number + 1):
        # 1과 자기 자신은 항상 약수이다.
        divisor_count = 2

        # 약수는 짝을 이루므로 제곱근까지만 확인한다.
        for i in range(2, int(n**(1/2)) + 1):
            if n % i == 0:
                # i가 약수이면 n // i도 약수이다.
                divisor_count += 1

                # 제곱수인 경우 같은 약수를 두 번 세지 않는다.
                if i**2 != n:
                    divisor_count += 1

        if divisor_count > limit:
            weight += power
        else:
            weight += divisor_count

    return weight
```

- Python 코드는 `2`번 기사부터 순회하고, `1`번 기사의 무게는 `weight = 1`로 미리 반영한다.
- 각 숫자 `n`에 대해 `1`과 자기 자신을 먼저 약수로 세고, `2`부터 `sqrt(n)`까지만 확인한다.
- `i`가 `n`의 약수라면 짝이 되는 약수 `n // i`도 함께 존재하므로 약수 개수를 `2`개 늘릴 수 있다.
- 단, `n`이 제곱수이면 `i`와 `n // i`가 같기 때문에 한 번만 더한다.
- 약수 개수가 `limit`을 넘으면 `power`를 더하고, 아니면 약수 개수를 더한다.
