### Intuition

---

- 문자열은 `n`번째 문자를 기준으로 오름차순 정렬해야 한다.
- 단, `n`번째 문자가 같은 문자열끼리는 문자열 전체를 사전순으로 정렬해야 한다.
- 따라서 정렬 기준은 다음 두 가지로 정리할 수 있다.
  - 1순위: `n`번째 문자
  - 2순위: 문자열 전체
- `n`번째 문자를 key로 사용해 문자열을 그룹화하고, key가 자동 정렬되는 `TreeMap`을 사용해 1순위 정렬을 처리한다.
- 같은 key를 가진 문자열들은 `Collections.sort()`로 다시 정렬해 2순위 조건을 만족시킨다.

<br>

### Approach

---

- `TreeMap<String, List<String>>`을 생성한다.
  - `TreeMap`은 key를 오름차순으로 유지하므로 `n`번째 문자를 기준으로 정렬된 순서를 얻을 수 있다.
  - value는 같은 `n`번째 문자를 가진 문자열들을 담기 위해 `List<String>`으로 둔다.
- `strings` 배열을 순회한다.
  - 각 문자열 `s`에서 `s.substring(n, n + 1)`로 `n`번째 문자를 추출한다.
  - 해당 key에 연결된 리스트를 가져오고, 없으면 새 리스트를 만든다.
  - 리스트에 현재 문자열을 추가한 뒤 map에 저장한다.
- `TreeMap`의 value들을 순서대로 순회한다.
  - 각 리스트를 `Collections.sort()`로 사전순 정렬한다.
  - 정렬된 문자열들을 차례대로 `answer` 배열에 넣는다.
- 완성된 `answer` 배열을 반환한다.

<br>

### Complexity

---

- Time complexity: `O(m log k + m log m)`
  - `m`은 문자열 개수, `k`는 서로 다른 `n`번째 문자의 개수이다.
  - `TreeMap`에 삽입할 때 각 문자열마다 `O(log k)`가 든다.
  - 이후 각 그룹을 정렬하는 비용의 총합은 최악의 경우 `O(m log m)`이다.

- Space complexity: `O(m)`
  - `TreeMap`과 각 리스트에 입력 문자열 참조를 저장한다.
  - 정답 배열도 문자열 개수만큼 필요하다.

<br>

### Code (Java)

---

```java
import java.util.*;

class Solution {
    public String[] solution(String[] strings, int n) {
        String[] answer = new String[strings.length];

        // Map 생성 (key 기준 정렬을 위해 TreeMap 사용)
        Map<String, List<String>> map = new TreeMap<>();

        // strings 순회하며 Map 구성
        for (String s: strings) {
            String k = s.substring(n, n+1);
            List<String> v = map.getOrDefault(k, new ArrayList<>());

            v.add(s);
            map.put(k, v);
        }

        int idx = 0;

        // Map을 순회하며 정답 배열에 저장
        for (List<String> v: map.values()) {
            // value를 오름차순으로 정렬
            Collections.sort(v);

            for (String s: v) {
                answer[idx++] = s;
            }
        }

        return answer;
    }
}
```

### Better Code (Java)

---

```java
import java.util.*;

class Solution {
    public String[] solution(String[] strings, int n) {
        Arrays.sort(strings, (a, b) -> {
            // n번째 문자가 서로 같으면 문자열 전체를 사전순으로 비교
            if (a.charAt(n) == b.charAt(n)) {
                return a.compareTo(b);
            }

            // n번째 문자가 서로 다르면 n번째 문자를 비교
            return Character.compare(a.charAt(n), b.charAt(n));
        });

        return strings;
    }
}
```

- 별도의 `Map`과 정답 배열을 만들지 않고, 배열 자체를 정렬할 수 있다.
- 비교 기준을 comparator에 직접 작성하면 문제 조건인 `n`번째 문자 우선 정렬과 문자열 전체 사전순 정렬을 한 번의 정렬로 처리할 수 있다.
- 시간 복잡도는 `O(m log m)`, 추가 공간 복잡도는 정렬 구현에 따른 보조 공간을 제외하면 `O(1)`에 가깝다.
