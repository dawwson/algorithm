> 풀이 날짜 : 2026.03.25  
> 문제 유형 : 그래프 탐색 (BFS/DFS)  
> 문제 제목 : 2606. 바이러스
> 문제 링크 : https://www.acmicpc.net/problem/2606

<br>

### Intuition

---

<!--
- 제약 조건
- 단순 접근의 한계
- 필요한 성질 (시간/공간/구조)
- 선택한 해법
-->

- 제약 조건:
  - 컴퓨터의 수 `N`이 최대 `100`으로 매우 작다. 시간 복잡도보다는 그래프의 연결 관계를 정확히 파악하는 것이 핵심이다.

- 구조:
  - 컴퓨터 간의 연결 관계는 정점(Computer)과 간선(Network)으로 이루어진 양방향 그래프로 모델링 가능하다.

- 핵심 성질:
  - 1번 컴퓨터가 바이러스에 걸렸을 때, 이와 연결된 모든 노드를 방문하여 개수를 세어야 한다.
  - 즉, 동일한 연결 컴포넌트에 속한 노드를 탐색하는 문제이다.

- 선택한 해법:
  - 시작 노드에서 인접한 노드부터 차례대로 방문하는 BFS(너비 우선 탐색)를 사용하여 전염 경로를 추적한다.

<br>

### Approach

---

<!--
- 실제로 채택한 알고리즘
- 자료구조 선택 이유
- 단계별 처리 방식
-->

- 알고리즘:
  - 큐(`Queue`) 자료구조를 활용한 BFS를 선택했다. 시작점인 1번 노드부터 너비 우선 탐색을 진행하며 방문하지 않은 연결 노드를 찾는다.

    (+ 이 문제의 경우 연결 컴포넌트(연결된 덩어리)만 파악하면 되기 때문에 `BFS`, `DFS` 중 어떤 것으로 풀어도 상관은 없다. )

- 자료구조 선택 이유:
  - `List<List<Integer>>` (인접 리스트):
    - 정점 수에 비해 간선 수가 적은 희소 그래프에서 메모리 효율적이며, 특정 노드의 인접 노드를 순회하는 속도가 빠르다.
  - `boolean[] visited`:
    - 이미 방문한 컴퓨터를 중복 방문하여 무한 루프에 빠지는 것을 방지한다.
  - `ArrayDeque`:
    - `Java`의 `LinkedList`보다 큐 연산 속도가 빨라 성능상 유리하다.

- 단계별 처리 방식:
  1. `N+1` 크기로 그래프와 방문 배열을 초기화한다.
  2. 입력받은 간선 정보를 바탕으로 양방향 인접 리스트를 생성한다.
  3. 1번 노드를 큐에 넣고 방문 처리한다.
  4. 큐에서 노드를 하나씩 꺼내며 연결된 인접 노드 중 미방문 노드를 큐에 삽입하고 `count`를 증가시킨다.

<br>

### Complexity

---

- Time complexity: `O(N + M)`
  - 정점의 개수를 `N`, 간선의 개수를 `M`이라고 할 때, 그래프를 초기화하고 입력하는 과정에서 `O(N + M)`이다.
  - 그래프 탐색 과정에서 모든 정점을 한 번씩 방문하고, 모든 간선을 두 번씩 확인(양방향 그래프)하므로 `O(N + M)`이다.

- Space complexity: `O(N + M)`
  - 인접 리스트 저장 공간(`O(N+M)`)과 방문 배열(`O(N)`), 큐의 최대 크기(`O(N)`)를 고려한 공간 복잡도이다.

<br>

### Code (Java)

---

```java
import java.util.*;
import java.lang.*;
import java.io.*;

class Main {
    static List<List<Integer>> graph;
    static boolean[] visited;
    static int count = 0;

    public static void main(String[] args) throws Exception {
        // 입출력 스트림 생성
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        StringTokenizer st;

        // 컴퓨터의 수 (= 번호)
        int N = Integer.parseInt(br.readLine());
        // 간선의 수
        int M = Integer.parseInt(br.readLine());

        // 그래프 생성
        graph = new ArrayList<>();
        for (int i = 0; i <= N; i++) {
            graph.add(new ArrayList<>());
        }
        visited = new boolean[N + 1];

        for (int i = 0; i < M; i++) {
            st = new StringTokenizer(br.readLine());

            int U = Integer.parseInt(st.nextToken());
            int V = Integer.parseInt(st.nextToken());

            // 양방향 연결
            graph.get(U).add(V);
            graph.get(V).add(U);
        }

        bfs(1);

        System.out.print(count);
    }

    public static void bfs(int start) {
        Queue<Integer> queue = new ArrayDeque<>();

        // 시작 노드 방문 처리
        queue.offer(start);
        visited[start] = true;

        while (!queue.isEmpty()) {
            int current = queue.poll();

            // 인접 노드 탐색
            for (int next : graph.get(current)) {
                // 아직 방문하지 않았다면 큐에 넣고 방문 처리
                if (!visited[next]) {
                    queue.add(next);
                    visited[next] = true;
                    count += 1;
                }
            }
        }
    }
}
```
