### Intuition

---

<!--
- 제약 조건
- 단순 접근의 한계
- 필요한 성질 (시간/공간/구조)
- 선택한 해법
-->

- 제약 조건:
  - 지방의 수 `N`은 최대 10,000
  - 총 예산 `M`은 최대 1,000,000,000(10억)이다.

- 단순 접근의 한계:
  - 상한액을 1원부터 가능한 최대치까지 하나씩 확인하는 선형 탐색(`O(M)`)은 예산 범위가 너무 커서 시간 초과가 발생할 수 있다.

- 필요한 성질:
  - 상한액이 높아질수록 배정되는 총 예산 합계가 증가하므로, 특정 지점을 기준으로 가능/불가능이 나뉘는 구조이다.

- 선택한 해법:
  - "조건을 만족하는 최댓값"을 찾기 위해 탐색 범위를 절반씩 줄여나가는 **이진 탐색**을 선택했다.
  - 최적화 문제를 결정 문제로 바꾸어 푸는 파라메트릭 서치(Parametric Search)를 적용하여 탐색 효율을 극대화한다.
    - 최적화 문제: "전체 예산을 넘지 않는 가장 큰 상한액은 얼마인가?"  
      (답이 딱 하나인 어려운 문제)
    - 결정 문제: "상한액을 XXX원으로 잡으면, 전체 예산을 넘어 안 넘어?"  
      (답이 YES/NO로 나오는 쉬운 문제)

<br>

### Approach

---

<!--
- 실제로 채택한 알고리즘
- 자료구조 선택 이유
- 단계별 처리 방식
-->

- 알고리즘: 이진 탐색을 응용하여 가능한 최대 상한액을 결정한다.

- 단계별 처리 방식:
  1. 입력받은 예산 요청액 중 최댓값을 구하여 탐색 범위의 끝(high)으로 설정한다.
  2. `low = 0`, `high = 최대 요청액`으로 설정하고 이진 탐색을 시작한다.
  3. 매 단계에서 `mid`(상한액 후보)를 정하고, 이에 따른 실제 예산 배정 총합을 계산한다.
  4. 배정 총합이 전체 예산 `M` 이하인 경우, 현재 `mid`를 `answer`에 저장하고 더 큰 상한액을 찾기 위해 `low`를 높인다.
  5. 배정 총합이 `M`을 초과하는 경우, 상한액이 너무 높으므로 `high`를 낮춘다.

<br>

### Complexity

---

- Time complexity: `O(N * log(max_budget))`
  - 상한액의 범위를 `0`부터 예산 요청액의 최댓값이라고 할 때, 이진 탐색은 한 번 확인할 때마다 탐색 범위가 절반씩 줄어든다. 즉, `while` 문이 돌아가는 전체 횟수는 예산의 최댓값에 로그를 취한 `O(log(max_budget))`이 된다.
  - `while` 문이 한 번 돌 때마다 `mid`가 조건에 맞는지 검사하기 위해 모든 지방의 예산을 하나씩 꺼내서 `mid`와 비교한다. 이 과정에서 `for` 문이 N번 돌아가므로 `O(N)`의 시간이 걸린다.
  - `while` 문이 log(max_budget)번 실행되는데, 그 안에서 매번 `O(N)`짜리 작업을 수행하므로 `O(N * log(max_budget))`이 된다.

- Space complexity: `O(N)`
  - `N`개의 예산 요청액을 저장하기 위한 정수 배열을 사용한다.

### Code (Java)

---

1️⃣ **이진 탐색**

```java
import java.util.*;
import java.lang.*;
import java.io.*;

class Main {
    public static void main(String[] args) throws Exception {

        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        StringTokenizer st = new StringTokenizer(br.readLine());

        // 예산 요청 개수
        int N = Integer.parseInt(st.nextToken());
        // 예산 요청 금액
        int[] budgets = new int[N];
        // 가능한 총 예산
        int M = 0;

        st = new StringTokenizer(br.readLine());
        for (int i = 0; i < N; i++) {
            budgets[i] = Integer.parseInt(st.nextToken());
        }

        st = new StringTokenizer(br.readLine());
        M = Integer.parseInt(st.nextToken());

        Arrays.sort(budgets);

        // 요청된 예산의 중간값을 상한액의 시작 금액으로 설정
        int low = 0;
        int high = budgets[N - 1];
        int answer = 0; // 상한액

        while (low <= high) {
            int mid = (low + high) / 2;
            int sum = 0;

            for (int i = 0; i < N; i++) {
                if (budgets[i] > mid) {
                    sum += mid;
                } else {
                    sum += budgets[i];
                }
            }

            if (sum <= M) {
                answer = mid;
                low = mid + 1;
            } else {
                high = mid - 1;
            }
        }

        System.out.print(answer);
    }
}
```

2️⃣ **그리디**

```java
Arrays.sort(budgets);
long remainingBudget = M;
int remainingDepts = N;
int answer = 0;

for (int i = 0; i < N; i++) {
    // 현재 남은 예산을 남은 지자체 수로 나눈 '평균 상한액'
    int avg = (int) (remainingBudget / remainingDepts);

    if (budgets[i] <= avg) {
        // 요청액이 평균보다 작으면 그대로 승인
        remainingBudget -= budgets[i];
        remainingDepts--;
        answer = budgets[i]; // 현재까지의 최대 배정액 갱신
    } else {
        // 요청액이 평균보다 크면, 남은 예산을 공평하게 나누고 종료
        answer = avg;
        remainingBudget = 0;
        break;
    }
}

// 만약 예산이 남아서 모든 요청을 다 들어준 경우,
// answer는 자동으로 가장 큰 요청액(budgets[N-1])이 됨
```
