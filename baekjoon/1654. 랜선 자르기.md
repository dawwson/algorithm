> 문제 제목 : 1654. 랜선 자르기
> 문제 링크 : https://www.acmicpc.net/problem/1654

### Intuition

---

<!--
- 제약 조건
- 단순 접근의 한계
- 필요한 성질 (시간/공간/구조)
- 선택한 해법

- 제약 조건:
  - 이미 가지고 있는 랜선의 개수 `K`는 최대 10,000개, 필요한 랜선의 개수 `N`은 최대 `1,000,000`개이다.
  - 각 랜선의 길이는 `2^31-1` 이하의 자연수이다.

- 단순 접근의 한계:
  - `1`부터 가장 긴 랜선의 길이까지 하나씩 확인하며 최댓값을 찾는 브루트포스(`Brute-force`) 방식은 시간 복잡도상 불가능하다 (`2^31`번의 연산 필요).

- 필요한 성질:
  - 특정 길이 `L`로 잘랐을 때 나오는 랜선의 개수 `f(L)`은 `L`이 커질수록 작아지는 성질을 가진다.

- 선택한 해법:
  - 정답이 될 수 있는 '길이'의 범위를 좁혀가며 최적의 값을 찾는 이진 탐색(Binary Search), 그중에서도 조건을 만족하는 최댓값을 찾는 `Upper Bound`(특정 값보다 큰 값이 처음으로 나타나는 위치를 찾는 알고리즘) 개념을 적용한다.

<br>

### Approach

---

<!--
- 실제로 채택한 알고리즘
- 자료구조 선택 이유
- 단계별 처리 방식
-->

- 알고리즘:
  - 이진 탐색을 사용하여 `1`부터 `cables[K-1]` 사이에서 최적의 길이를 찾는다.

- 자료구조 선택:
  - `int[] cables`: 각 랜선의 길이를 저장한다.
  - `long`: 이진 탐색의 `low`, `high`, `mid` 및 `sum` 계산 시 `Integer Overflow`를 방지하기 위해 `long` 타입을 사용한다.

    (특히 `low + high`가 `2^{31}`을 넘을 수 있음)

- 단계별 처리 방식:
  1. 입력받은 랜선들을 정렬하여 가장 긴 랜선의 길이를 확인한다.
  2. `low = 1`, `high = max_cable_length`로 설정한다.
  3. `mid` 길이로 모든 랜선을 잘라 총 몇 개(`sum`)가 나오는지 계산한다.
  4. `sum >= N`이면, 일단 조건은 만족하므로 answer에 현재 `mid`를 저장하고, 더 긴 길이가 가능한지 확인하기 위해 `low`를 높인다.
  5. `sum < N`이면, 너무 길게 자른 것이므로 `high`를 줄여 길이를 짧게 만든다.

<br>

### Complexity

---

- Time complexity: `O(K log L)`
  - 랜선의 최대 길이(`2^31 - 1`)를 `L`, 가지고 있는 랜선의 개수를 `K`라고 하자.
  - 이진 탐색에서 `log L`이 소요되고, 각 단계에서 `K`개의 랜선을 순회한다.

- Space complexity: `O(K)`
  - `K`개의 랜선 길이를 저장하기 위한 배열 공간이 필요하다.

<br>

### Code (Java)

---

```java
import java.util.*;
import java.lang.*;
import java.io.*;

class Main {
    public static void main(String[] args) throws Exception {

        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        StringTokenizer st = new StringTokenizer(br.readLine());

        // 가지고 있는 랜선 개수
        int K = Integer.parseInt(st.nextToken());
        // 필요한 랜선 개수
        int N = Integer.parseInt(st.nextToken());
        // 랜선 길이
        int[] cables = new int[K];

        for (int i = 0; i < K; i++) {
            cables[i] = Integer.parseInt(br.readLine());
        }

        // 오름차순 정렬
        Arrays.sort(cables);

        long low = 1;
        long high = cables[K - 1];

        // 최대 랜선의 길이
        long answer = 0;

        while (low <= high) {
            long mid = (low + high) / 2;

            // 랜선 개수 합계
            long sum = 0;

            for (int cable : cables) {
                sum += cable / mid;
            }

            if (sum < N) {
                // 필요한 개수보다 적으면 상한선을 내림
                high = mid - 1;
            } else if (sum >= N) {
                // 필요한 개수보다 많으면 하한선을 올림
                low = mid + 1;
                answer = mid;
            }
        }

        System.out.print(answer);
    }
}
```
