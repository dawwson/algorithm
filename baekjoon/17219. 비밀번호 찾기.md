> 풀이 날짜 : 2026.01.03  
> 문제 유형 : 해시 테이블  
> 문제 제목 : 17219. 비밀번호 찾기  
> 문제 링크 : https://www.acmicpc.net/problem/17219

<br>

### Intuition

---

- 문제의 핵심 요구사항은

  - 사이트 주소(String)가 주어졌을 때
  - 해당하는 비밀번호(String)을 빠르게 조회하는 것이다.

- 입력 크기는

  - 사이트 개수 `N <= 100,000`
  - 비밀번호를 찾으려는 사이트 주소의 수 `M <= 100,000`
  - 최악의 경우 `O(N * M)`이 되어 시간 초과가 발생할 수 있다.

- 각 질의를 상수 시간에 처리할 수 있는 자료구조가 필요하다.
  - 입력과 출력 모두 문자열 기반으로 문자열 `Key` 조회에 최적화된 구조가 요구된다.
  - 따라서 해시 테이블 기반의 `Map`이 문제 요구사항과 정확히 일치한다고 판단했다.

<br>

### Approach

---

- 알고리즘
  - 해시 테이블 기반 조회
- 자료구조

  - `HashMap<String, String>`은 `Key` 기반 조회를 평균 `O(1)`에 수행 가능하고 문자열 `Key`에도 안정적인 성능을 제공한다.

- 처리 단계
  - `N`개의 사이트 주소와 비밀번호를 `Map`에 저장한다.
  - `M`개의 질의에 대해
    - 입력된 사이트 주소(`Key`)로 사용해서 `Map`에서 비밀번호(`Value`)를 조회한다.
  - 조회 결과를 `StringBuilder`에 누적 후 한 번에 출력한다.

<br>

### Complexity

---

- Time complexity: `O(N + M)`

  - 사이트 정보 저장에 `O(N)`, 질의 처리에 `O(M)`이다.
  - `put`, `set` 연산은 평균 `O(1)`이다.

- Space complexity: `O(N)`
  - 사이트 주소와 비밀번호를 저장하는 `Map` 사용

<br>

### Code (Java)

---

```java
import java.util.*;
import java.lang.*;
import java.io.*;

class Main {
    public static void main(String[] args) throws Exception {

        // 입출력 스트림 생성
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        StringBuilder sb = new StringBuilder();

        // 첫 줄 읽기
        StringTokenizer st = new StringTokenizer(br.readLine());
        int N = Integer.parseInt(st.nextToken());
        int M = Integer.parseInt(st.nextToken());

        // { 사이드 주소 : 비밀번호 } 저장
        Map<String, String> map = new HashMap<>();

        for (int i = 0; i < N; i++) {
            st = new StringTokenizer(br.readLine());

            String address = st.nextToken();
            String password = st.nextToken();

            map.put(address, password);
        }

        // 비밀번호 조회
        for (int i = 0; i < M; i++) {
            String key = br.readLine();
            sb.append(map.get(key) + "\n");
        }

        // 출력
        System.out.print(sb);
    }
}
```
