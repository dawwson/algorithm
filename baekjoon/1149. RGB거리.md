> 문제 제목 : 1149. RGB 거리  
> 문제 링크 : https://www.acmicpc.net/problem/1149

### Intuition

---

<!--
- 제약 조건
- 단순 접근의 한계
- 필요한 성질 (시간/공간/구조)
- 선택한 해법
-->

- 제약 조건:
  - 인접한 집($i-1$번째와 $i$번째, $i$번째와 $i+1$번째)은 서로 같은 색을 칠할 수 없다.
  - 즉, 현재 집의 색은 이전 집의 색과 달라야 한다.

- 단순 접근의 한계:
  - 매 단계에서 가장 저렴한 비용을 선택하는 그리디(Greedy) 방식은, 현재의 저렴한 선택이 다음 집에서 엄청나게 비싼 비용을 강제할 수 있기 때문에 최적해를 보장하지 않는다.

- 필요한 성질:
  - $i$번째 집을 특정 색으로 칠했을 때의 최소 비용은, $i-1$번째 집을 그 색을 제외한 나머지 두 색으로 칠했을 때의 최적해에 의존한다. (최적 부분 구조)

- 선택한 해법:
  - 각 단계마다 '빨강, 초록, 파랑'으로 끝나는 세 가지 경우의 최적해를 모두 유지하며 나아가는 2차원 Bottom-up DP를 사용한다.

<br>

### Approach

---

<!--
- 실제로 채택한 알고리즘
- 자료구조 선택 이유
- 단계별 처리 방식
-->

- 채택한 알고리즘:
  - 2차원 다이나믹 프로그래밍 (Bottom-up)

- 자료구조 선택 이유:
  - 집의 번호($i$)와 선택한 색상($j$)이라는 두 가지 상태를 동시에 관리하기 위해 `int[N+1][3]` 2차원 배열을 사용한다.
  - 열 인덱스(0, 1, 2)가 곧 색상 정보를 대신하므로 별도의 저장 공간이 필요 없다.

- 단계별 처리 방식:
  1. 초기화: 첫 번째 집($i=1$)의 각 색상별 비용을 dp[1]에 그대로 저장한다.
  2. 반복 및 전이: 2번째 집부터 $N$번째 집까지 다음 로직을 반복한다.
     1. $i$번째를 빨강(0)으로 칠할 때: $i-1$번째의 초록(1)과 파랑(2) 중 최솟값을 가져와 현재 비용을 더한다.
     2. $i$번째를 초록(1)으로 칠할 때: $i-1$번째의 빨강(0)과 파랑(2) 중 최솟값을 가져와 현재 비용을 더한다.
     3. $i$번째를 파랑(2)으로 칠할 때: $i-1$번째의 빨강(0)과 초록(1) 중 최솟값을 가져와 현재 비용을 더한다.
  3. 최종 정답: $N$번째 집까지 계산이 끝난 후, `dp[N][0]`, `dp[N][1]`, `dp[N][2]` 중 가장 작은 값을 출력한다.

<br>

### Complexity

---

- Time complexity: $O(N)$
  - 집의 개수 $N$만큼 반복문을 1회 수행하며, 각 루프 내 연산은 상수 시간(3가지 색상 비교)이 소요된다.

- Space complexity: $O(N)$
  - $N * 3$ 크기의 `dp` 배열과 `costs` 배열을 사용한다.
    (입력값을 dp 배열에 바로 누적한다면 $O(1)$ 수준의 추가 공간으로도 최적화가 가능하다.)

<br>

### Code (Java)

---

```java
import java.util.*;
import java.lang.*;
import java.io.*;

class Main {
    public static void main(String[] args) throws Exception {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        StringTokenizer st = new StringTokenizer(br.readLine());

        // 집의 개수
        int N = Integer.parseInt(st.nextToken());
        // 각 집을 RGB로 칠하는 비용
        int[][] costs = new int[N+1][3];

        for (int i = 1; i <= N; i++) {
            st = new StringTokenizer(br.readLine());

            int r = Integer.parseInt(st.nextToken());
            int g = Integer.parseInt(st.nextToken());
            int b = Integer.parseInt(st.nextToken());

            costs[i][0] = r;
            costs[i][1] = g;
            costs[i][2] = b;
        }

        // i번째 집을 R(0)G(1)B(2)로 칠할 때의 누적 최소 비용
        int[][] dp = new int[N+1][3];
        dp[1][0] = costs[1][0];
        dp[1][1] = costs[1][1];
        dp[1][2] = costs[1][2];

        // 점화식
        for (int i = 2; i <= N; i++) {
            dp[i][0] = Math.min(dp[i-1][1], dp[i-1][2]) + costs[i][0];
            dp[i][1] = Math.min(dp[i-1][0], dp[i-1][2]) + costs[i][1];
            dp[i][2] = Math.min(dp[i-1][0], dp[i-1][1]) + costs[i][2];
        }

        Arrays.sort(dp[N]);

        System.out.print(dp[N][0]);
    }
}
```
